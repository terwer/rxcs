# AutoJS6 综述

## 它是什么

AutoJS6 是一个运行在 Android 平台上的 JavaScript 自动化工具。

从官方文档的定位看，它同时承担三类角色：

- 自动化执行器
- JavaScript 脚本运行环境
- 带一定编辑能力的脚本开发工具

## 语言与运行时

官方综述里明确了几件事：

- 语言：JavaScript
- 引擎：Rhino
- ES 支持：ES5 全量，ES6 部分

这意味着：

- 你写的是原生 JS 风格代码，不再受按键精灵 VB 语法限制
- 但也不能默认把它当完整现代 Node.js 环境
- 兼容性要按 Rhino 和 AutoJS6 提供的能力来判断

## 它能做什么

官方综述提到的能力可以分成两大类。

### 自动化能力

- 自动化操作
- 控件操作
- 应用操作
- 图像处理
- 文字识别
- UI 交互
- 对话框
- 悬浮窗
- 画布
- 多线程
- 协程
- 异步编程
- 事件监听
- 定时任务
- HTTP 请求
- Shell

### 工具能力

- 设备信息
- 传感器信息
- 控件信息
- Base64 编解码
- 密文生成
- 数学运算
- 颜色转换

## 为什么适合你后面的新项目

结合当前 RXCS 项目的目标，AutoJS6 适合做游戏自动化，主要不是因为“API 更多”，而是因为下面三点：

- 语法是 JS，本身更适合抽象和拆模块
- 控件、图片、动作、UI 可以分层写
- 调试方式比按键精灵手机版更容易工程化

## 如何阅读官方文档

官方文档自己也强调了几个阅读点：

- 文档中的章节通常围绕某个对象展开
- 对象后续的方法和属性可能省略对象名前缀
- 参数类型、返回值类型、示例代码是阅读重点

把它翻成工程语言就是：

- 先确认“当前章节的临时作用域对象是谁”
- 再看这个对象能做哪些动作
- 最后再看参数约束和示例

## 阅读优先级建议

如果是做你后面的 `rxcs` 新项目，第一轮建议优先读这些官方章节：

1. `Automator`
2. `UiSelector`
3. `UiObject`
4. `UiObjectActions`
5. `Image`
6. `App`
7. `Device`
8. `Floaty`
9. `UI`
10. `Thread`

## 当前理解边界

官方 `Automator`、`UI` 等章节里有一部分内容仍标了“待补充”，说明：

- 官方文档足够覆盖主线能力
- 但不能指望每个章节都已经像成熟 SDK 一样完整

所以后续沉淀方式应该是：

- 先用官方文档确定能力边界
- 再结合你自己的稳定项目，把“怎么落地”写成项目内部文档
