## Why

登录能力已经跑通，RXCS 当前已经具备从启动到进入登录后状态的稳定入口。下一步必须尽快把“进入区服选择页 → 点击绿色版入口 → 选择绿色一区”这条链路建立起来，否则后续首页、日常任务等流程仍然没有稳定落点。

现在推进这项变更，是为了把登录后的第一段业务路径固定下来，让项目从“能登录”推进到“能进入正确区服”，为后续主页识别和任务流打基础。

## What Changes

- 新增 RXCS 选区能力的正式规格，明确选区页识别、绿色版入口点击、绿色一区选择、结果校验与失败处理规则。
- 定义选区流程的原子测试模式，要求先验证页面 ready、绿色版入口、绿色一区选择，再允许整合 full flow。
- 规定选区流程优先使用可稳定识别的控件或页面特征；无法稳定控件化时，使用命名固定的图片资源兜底。
- 明确绿色版入口与绿色一区所需的资源截图命名规范，便于后续补齐图片并稳定实现。

## Capabilities

### New Capabilities
- `rxcs-server-select-flow`: 规范 RXCS 登录后选区页的识别、绿色版入口触发、绿色一区选择、结果校验与调试模式行为。

### Modified Capabilities
- 无

## Impact

- 影响代码范围：`modules/` 下的 RXCS 业务模块与后续登录后主流程衔接逻辑。
- 影响资源范围：绿色版入口、绿色一区、选中态或进入成功态等选区相关图片资源。
- 影响调试流程：测试入口将新增选区原子测试模式，而不是直接从登录跳到任务流。
- 影响后续能力：主页识别、任务流、区服状态判断都将依赖本次选区能力作为前置入口。
