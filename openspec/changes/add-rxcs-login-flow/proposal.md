## Why

当前 RXCS 项目已经完成 AutoJS6 的基础项目初始化，但尚未建立稳定、可验证、可扩展的登录方案。此前按键精灵版本在登录页上暴露出输入框路由不稳定、调试粒度过粗、工具层问题掩盖业务问题等缺陷，因此需要在 AutoJS6 中重新定义一套以控件优先、图片兜底、分步校验、失败即停为原则的登录能力。

现在推进这项变更，是为了尽快把“启动游戏 → 处理启动弹窗 → 进入登录页 → 输入账号密码 → 勾选协议 → 点击登录”这条最小可用链路稳定下来，为后续选区、日常任务、状态校验等功能提供可靠入口。

## What Changes

- 新增 RXCS 登录能力的正式规格，明确登录页的识别条件、输入流程、协议处理规则、登录按钮触发规则与失败处理规则。
- 定义登录流程的调试模式，要求先支持原子级测试，再允许整合为完整登录流程。
- 规定登录页优先使用控件 `id` 进行识别和操作；仅在协议勾选状态与特定视觉目标上使用图片资源兜底。
- 规定登录按钮与协议状态所需的资源截图命名规范，便于后续补齐资源并稳定落地实现。
- 明确 AutoJS6 登录流程必须遵守“每步校验、失败即停、禁止绕过内置输入流程”的工程约束。

## Capabilities

### New Capabilities
- `rxcs-login-flow`: 规范 RXCS 登录页的识别、输入、协议处理、按钮点击、调试模式与结果校验行为。

### Modified Capabilities
- 无

## Impact

- 影响代码范围：`main.js`、`modules/` 下的 RXCS 业务模块与公共动作模块。
- 影响资源范围：`assets/agree_no.png`、`assets/agree_yes.png`、`assets/login_btn.png` 及后续登录相关图片资源。
- 影响调试流程：测试入口将以登录原子测试和 guarded flow 为主，而不是直接运行整条业务链路。
- 影响后续能力：选区、任务流、登录结果校验等功能都将依赖本次登录能力作为前置入口。
